# Code of conduct

sharp-mud is presently a solo/small-collaborator project. Keep this light
until there's an actual outside contributor base, but hold the line on:

- Be direct and specific in review feedback — point at the line and the
  concrete failure mode, not vague style preferences.
- No drive-by rewrites bundled into an unrelated change — if you see
  something worth fixing while working on something else, call it out
  separately (an issue, a follow-up PR) rather than inflating the current
  diff.
- Disagreements about architecture get resolved by writing the alternatives
  down (see `design-decisions.md`) and picking one with a stated reason —
  not by silently overriding a previous decision in a later PR.

If/when this project takes on outside contributors, promote this section
into a standalone `CODE_OF_CONDUCT.md` (e.g. adapt Contributor Covenant)
rather than growing it in place here.
